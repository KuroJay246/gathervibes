# Phase 21 Event Command Center Options

This note preserves the next practical options after the Phase 21 command-center pass.

## 1. Event readiness checklist

- Can use existing data now:
  - selected Working Event
  - registrations
  - ticket assignment status
  - check-in progress
  - operations ledger summary
- No new rules or collections required for a first version.

## 2. Vendor / sponsor / baker tracker

- Partly possible with existing `operationsLedger` categories and notes.
- A real tracker would eventually need a clearer data model for contacts, statuses, and follow-up fields.

## 3. Supplies and print-materials tracker

- Not a good fit for the current data model alone.
- Would need a future data model review before implementation.

## 4. Payment reconciliation review

- Can go further with existing registration finance fields and operations ledger summaries.
- No payment gateway is required for a review-only workflow.

## 5. Post-event report

- Can start with existing data:
  - registration totals
  - guest totals
  - payment summary
  - ticket/check-in progress
  - operations net summary
- A richer version could later add structured event notes.

## 6. Event template / setup checklist

- Partly possible with existing event defaults and dashboard guidance.
- A complete reusable checklist likely needs a future saved-template model.

## 7. Communications follow-up packets

- Can use existing communications copy-only tools now.
- Automatic sending remains out of scope and not live.

## 8. Staff / access workflow later

- Requires future approved workflow phases.
- Do not use this phase to activate requester submit, approval, decline, revoke, or assignment editing.

## 9. Scanner only when event day is near

- Scanner is already live for its current limited role.
- Additional scanner expansion should wait for real event-day need, not drive the organizer/admin command-center roadmap.
