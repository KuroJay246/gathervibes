# New Event Setup Guide

## Open the planner

1. Open `Events`.
2. Select `Plan a New Event`.
3. Complete the planner from top to bottom. You can edit the event later if details change.

## Step 1 - Event Basics

Add the core event details:

- event name
- event type
- event date
- start time
- end time
- venue
- location
- short event description
- event status

Use `Draft` when the idea is still forming and `Planning` when the event is ready for normal organizer work.

## Step 2 - Capacity and Registration

Set the guest and registration basics:

- expected capacity
- whether registration is required
- number of ticket types
- default ticket price
- complimentary attendees allowed
- door payment allowed
- registration opening date
- registration closing date

Use price tiers when you need different ticket prices. Use the default ticket price when one price is enough.

## Step 3 - Financial Plan

Add the planning numbers that help the organizer prepare cash needs:

- projected registration income
- venue budget
- supplier budget
- entertainment budget
- marketing budget
- staffing budget
- contingency
- other budget

These values support planning. They do not replace real payment records in `Payments` or real event-level records in `Operations`.

## Step 4 - Operations

Capture the setup notes needed to run the event:

- venue access time
- suppliers note
- baker and vendor note
- sponsor note
- staff or helper note
- equipment note
- licence note
- insurance note
- setup time
- event-day timeline
- emergency contact

You can add full partner and commitment records after the event is created.

## Step 5 - Readiness

Review the checklist before saving:

- confirm missing essentials
- mark items already ready
- leave optional items alone when they are not required
- create the event

After save, the new event becomes the Working Event and opens into the organizer workflow.

## After the event is created

1. Review `Overview`.
2. Add planning tasks in `Events`.
3. Add registrations and payments.
4. Add supplier, baker, sponsor, and staff records in `Operations`.
5. Return to `Reports` for the read-only summary and follow-up view.
